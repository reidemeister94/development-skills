# Prompt hygiene — full check

Invoked from `SKILL.md` Phase B step 2.b, immediately after the user selects or drafts
the `prompt_file`.

When the prompt itself tells the agent to re-run `measure_cmd` or profile iteratively,
the agent correctly follows the prompt — and correctly violates the harness protocol.
This is a prompt-design bug, not a model-discipline bug. Catch it here, before launching
trials.

Observed failure mode: a 4-hour `codex` session that spent most of its wall time
duplicating `measure_mgoa_wall.py` runs, step-timer profiling, and `cProfile` analysis,
because the prompt file explicitly instructed "use the harness tool for iterative
comparisons".

---

## Grep patterns

Run these on the selected `prompt_file`:

```bash
# 1. Extract basenames from every sub-command of gate_cmd and measure_cmd.
#    Splits on shell separators (&& || ; |), skips env-var prefixes (VAR=value)
#    and flag tokens (-x), takes the script path when the first token is an
#    interpreter (python, node, bash, ...). Same logic used by run_trial.py's
#    extract_command_basenames().
split_basenames() {
  printf '%s' "$1" | tr '&|;' '\n' | awk '
    {
      sub(/^[[:space:]]+/, ""); sub(/[[:space:]]+$/, "")
      primary = ""
      for (i = 1; i <= NF; i++) {
        if ($i ~ /^-/) continue
        if ($i ~ /=/ && $i !~ /^\.\// && $i !~ /^\//) continue
        primary = $i; start = i + 1; break
      }
      if (primary == "") next
      n = split(primary, parts, "/"); base = parts[n]
      if (base == "python" || base == "python3" || base == "node" || base == "bash" || base == "sh") {
        for (j = start; j <= NF; j++) {
          if ($j ~ /^-/) continue
          if ($j ~ /=/ && $j !~ /^\.\// && $j !~ /^\//) continue
          m = split($j, p2, "/"); print p2[m]; break
        }
      } else {
        print base
      }
    }' | awk 'NF' | sort -u
}

for bn in $(split_basenames "$gate_cmd") $(split_basenames "$measure_cmd"); do
  grep -inF "$bn" <prompt_file> || true
done

# 2. Profiling / benchmarking knobs that should be harness-only.
grep -inE '(PROFILE|PROFILING|STEP_TIMER|BENCHMARK|--runs |--warmup |hyperfine|pyinstrument|cProfile)' <prompt_file>

# 3. Rerun-the-full-harness phrases.
grep -inE '(run the (full )?regression|rerun (the )?(benchmark|measurement|tests? before and after)|iterative(ly)? (measure|profile|benchmark|compare))' <prompt_file>
```

Examples of what #1 now catches that the old `awk '{print $NF}'` missed:
- `gate_cmd = "./gradlew check && ./scripts/full_regression.sh"` → greps for
  `gradlew` AND `full_regression.sh` (previously only the second one).
- `gate_cmd = "ruff check . && mypy . && pytest -q"` → greps for all three.
- `measure_cmd = "ASSIGNMENT_STEP_TIMER=/tmp/x python scripts/bench.py --runs 15"`
  → greps for `bench.py` (previously matched `15`).

If any match fires, the check fails. Do not proceed silently.

**Note:** `run_trial.py` writes the computed basenames to
`<run_dir>/harness_commands.json` at launch. Step 2.b runs before that file exists,
so the wizard applies `split_basenames` directly against the TOML values in memory.
The downstream `parse_transcript.py` detector (for in-session violations) reads
`harness_commands.json` to cross-check what the agent actually ran.

## On match — user options

Display the offending lines in plain text:

> **Prompt hygiene warning**: your prompt contains instructions that tell the agent to
> run measurement/profiling/regression itself. The harness already owns baseline, post,
> and gate phases — the agent re-running them causes:
>
> - duplicate benchmark work (trial wall time inflates >2×)
> - methodology drift (the agent's ad-hoc measurements are not the harness's)
> - agent wall time exceeding the 60–150 min guidance
>
> Offending lines:
> `<file>:<line>: <matched text>`
>
> Options:
>   1. I'll edit the prompt now (remove the iterative-measurement guidance).
>   2. Rewrite with me — I'll propose a redacted version.
>   3. Override — type the exact sentence: `"I accept that the prompt contains measurement or profiling instructions that contradict the harness protocol: <reason>"`

Record override (if chosen) as `prompt_hygiene_override = "<full sentence>"` in
`.ai-agent-bench.toml`. The report will surface it as `prompt-hygiene-override`.

## Extension — adding language- or project-specific patterns

If your prompts routinely reference domain-specific measurement tools not covered above,
add their basenames to the first grep. Keep the regex in English — for non-English
prompts, either rewrite the prompt in English (recommended for portability) or add
language-specific patterns in a project-local extension. Do not edit this file inside the
plugin to add per-project idioms.
