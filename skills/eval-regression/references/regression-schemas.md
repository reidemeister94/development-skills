# Regression Eval Schemas

Quick reference for JSON structures used by eval-regression. For full skill-creator schemas, see `~/.claude/plugins/cache/claude-plugins-official/skill-creator/*/skills/skill-creator/references/schemas.md`.

---

## evals.json

Located at `<plugin>/evals/evals.json`.

```json
{
  "skill_name": "plugin-name",
  "evals": [
    {
      "id": 1,
      "name": "descriptive-kebab-case-name",
      "category": "category-name",
      "tests_change": "Which behavioral aspect this tests",
      "prompt": "The user's task prompt — realistic and specific",
      "expected_output": "Human-readable description of correct behavior",
      "files": [
        {
          "path": "relative/path/file.ext",
          "content": "File content provided as eval context"
        }
      ],
      "assertions": [
        {
          "name": "assertion-kebab-name",
          "type": "behavioral",
          "check": "What this assertion verifies",
          "pass_criteria": "How to determine pass/fail from the transcript"
        }
      ]
    }
  ]
}
```

### Writing good evals

- **Prompts**: Use realistic user requests, not abstract descriptions. Include specifics (file names, sizes, constraints).
- **Assertions**: Each must be objectively verifiable from the transcript. "Model invokes X" is good. "Model produces good output" is not.
- **Categories**: Group related evals. Common categories: routing, guard-conditions, anti-rationalization, workflow-phases, language-detection.
- **Coverage**: Test both positive cases (skill should do X) and negative cases (skill should NOT do Y).

## grading.json

Written per-run at `<eval-dir>/<version>/grading.json`.

```json
{
  "expectations": [
    {
      "text": "Description of what was checked",
      "passed": true,
      "evidence": "Specific quote or observation from transcript"
    }
  ],
  "summary": {
    "passed": 3,
    "failed": 1,
    "total": 4,
    "pass_rate": 0.75
  }
}
```

**Field names are viewer-strict**: `text`, `passed`, `evidence`. Not `name`/`met`/`details`.

## iteration_metadata.json

Written at `<workspace>/iteration-<N>/iteration_metadata.json`.

```json
{
  "iteration": 10,
  "old_version": "9.0.4",
  "new_version": "9.0.5",
  "timestamp": "2026-03-23T14:30:00Z",
  "changed_files": ["skills/core-dev/SKILL.md", "shared/workflow.md"],
  "evals_count": 26,
  "regression_summary": {
    "regressions": 0,
    "improvements": 2,
    "stable_pass": 22,
    "stable_fail": 2
  }
}
```

## benchmark.json

Generated by skill-creator's `aggregate_benchmark.py`. Uses `new_version`/`old_version` as configuration names instead of `with_skill`/`without_skill`.

```json
{
  "metadata": {
    "skill_name": "plugin-name",
    "timestamp": "2026-03-23T14:30:00Z",
    "evals_run": [1, 2, 3],
    "runs_per_configuration": 1
  },
  "runs": [
    {
      "eval_id": 1,
      "eval_name": "eval-name",
      "configuration": "new_version",
      "run_number": 1,
      "result": {
        "pass_rate": 1.0,
        "passed": 5,
        "failed": 0,
        "total": 5
      },
      "expectations": [
        {"text": "...", "passed": true, "evidence": "..."}
      ]
    }
  ],
  "run_summary": {
    "new_version": {
      "pass_rate": {"mean": 1.0, "stddev": 0.0, "min": 1.0, "max": 1.0}
    },
    "old_version": {
      "pass_rate": {"mean": 0.95, "stddev": 0.05, "min": 0.9, "max": 1.0}
    },
    "delta": {
      "pass_rate": "+0.05"
    }
  }
}
```

## regression-report.md

Human-readable regression analysis at `<workspace>/iteration-<N>/regression-report.md`. Template:

```markdown
# Regression Report: <plugin-name>
## <old_version> -> <new_version> | <date>

## Executive Summary
- Evals: N | Assertions: N
- Regressions: 0 | Improvements: 2
- Pass rate: 95% -> 100% (delta: +5%)
- Verdict: SAFE TO COMMIT

## Regressions
None.

## Improvements
| Eval | Assertion | Evidence |
|------|-----------|----------|
| eval-name | assertion-name | What changed and why it now passes |

## Per-Eval Comparison
| # | Eval | Old | New | Status |
|---|------|-----|-----|--------|
| 1 | brainstorm-guard-god-object | 5/5 | 5/5 | STABLE_PASS |
| 2 | brainstorm-guard-simple-typo | 2/3 | 3/3 | IMPROVEMENT |

## Changed Files
- skills/core-dev/SKILL.md (behavioral)
- shared/workflow.md (behavioral)
```
